# 🚀 EE5046 项目小组 Git 协作规范

## 1. 核心原则
* **main 分支禁止直接修改**：\`main\` 只存放稳定的版本。
* **严禁上传大文件**：数据集、模型权重必须留在 \`.gitignore\` 中。

## 2. 分支管理
* **每个人创建自己的分支**：\`git checkout -b 你的名字\`。
* **开发完成后发起 Pull Request**：由组长合并。

## 3. 路径规范
* **数据集使用公共路径**：不要拷贝到项目目录下。
* **确认 .gitignore 生效**：提交前检查 \`git status\`。
